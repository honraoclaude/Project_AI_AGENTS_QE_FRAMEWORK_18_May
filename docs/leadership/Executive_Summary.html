FSC Agentic QE Framework

AI-Powered Quality Engineering for FCA-Regulated Salesforce FSC Delivery
What We Built
The FSC Agentic QE Framework is a purpose-built AI pipeline that automatically applies 54 specialised quality engineering agents to every Jira user story — from initial refinement through production release. Designed for FCA-regulated Salesforce FSC Wealth Management delivery, it produces structured BDD acceptance criteria, FCA compliance checks, test design strategy, risk registers, and audit evidence in under 10 minutes per story. The framework is now enhanced with four game theory principles — adversarial AC challenge, confidence-weighted synthesis, Shapley attribution, and FCA-adaptive gate thresholds — making quality incentives mathematically rigorous across the delivery pipeline.

Business Problem — Across All Phases

  • Refinement: FCA obligations and AC gaps missed before development starts
  • Development: Apex coverage gaps, security vulnerabilities, and code quality issues found too late in the cycle
  • Testing: No structured CRT, UAT, or regression strategy — test gaps discovered during UAT execution
  • Release: Go/No-Go decisions made informally; FCA audit evidence assembled manually under pressure
  • 30+ hrs/sprint of senior QE time lost to documentation that should be automated

The Solution — End-to-End Coverage

  • Refinement: FCA classification, BDD acceptance criteria, adversarial AC challenge, Consumer Duty mapping, and risk register — before code starts
  • Development: AC compliance check, Apex coverage analysis (FCA-adaptive threshold), security scan, BDD Gherkin files, and test data architecture
  • Testing: CRT scenarios, UAT test cases, FCA regulatory tests, regression risk, defect triage, and flaky test detection — all structured and traceable
  • Release: Automated Go/No-Go gate, FCA Evidence Pack with Shapley attribution, change set integrity, dry-run, and production sign-off tracking

FCA Compliance — All Phases

  • COBS 9.2 (Refinement): Suitability gate classified and mandatory regulatory AC scenarios generated per story
  • Consumer Duty PS22/9 (Refinement + Testing): All four outcomes mapped at story level; vulnerable customer scenarios tested in CRT and UAT phases
  • FG21/1 §4.3 (Testing): Vulnerable customer pathways verified through dedicated FCA scenario agent
  • SYSC Audit Trail (Release): Immutable ledger across all phases; FCA Evidence Pack auto-assembled per story before release
  • Go/No-Go Gate (Release): Release blocked if FCA evidence pack is incomplete or unacknowledged
Game Theory Enhancements (New)
Minimax Cooperative Weighting Shapley Values Mechanism Design

AC Challenger — Adversarial Testing (Agent 05B)

  • Plays the role of a hostile FCA inspector against every BDD clause generated by Agent 05
  • Identifies five weakness types: ambiguous Given steps, non-observable Then outcomes, missing error paths, weak regulatory citations, untestable clauses
  • ACs that survive adversarial challenge are demonstrably stronger before development begins
  • Agent 09 (Risk Anticipation) adjusts confidence based on findings: 0 critical weaknesses = +8 confidence; 3+ = −10

Confidence Weighting, Shapley & Adaptive Gates

  • Confidence-weighted synthesis: Agent 09 weights upstream evidence by quality — a conf=85 FCA classification counts more than a conf=45 one
  • Shapley attribution: Fair marginal credit assigned to each upstream agent in the Risk Register and FCA Evidence Pack — mathematically transparent for FCA audit
  • Adaptive gates: HIGH-FCA stories face stricter thresholds (INVEST 85+, coverage 90%+); LOW-FCA stories fast-track at lower bars — correct incentives across the pipeline
